5. 메뉴 설계 근거

#설계 원칙설명
1대시보드 최상위 배치로그인 후 핵심 지표를 즉시 확인할 수 있도록 진입점으로 설정
2업무 프로세스 순서 반영상품 등록 → 주문 접수 → 배송 처리 흐름에 맞춰 메뉴 순서 배치
3정산은 후반부 배치결산 성격의 메뉴이므로 업무 흐름 후반부에 위치
4시스템 설정 최하단사용 빈도가 낮은 관리 메뉴는 최하단에 배치
5셀러 메뉴 축소 구조관리자 대비 축소된 메뉴로, 본인 데이터만 접근 가능
6AI 채팅 독립 패널모든 페이지에서 접근 가능한 우측 상시 패널로 분리
7로컬 LLM 전환 지원시스템 설정 > AI 설정에서 로컬 LLM / 외부 API 전환 가능
8예치금 선충전 모델셀러가 예치금을 선충전 후 주문 시 차감하는 구조로 결제 안정성 확보
9초대 코드 기반 가입관리자가 생성한 초대 코드/링크로만 셀러 가입 가능, 무분별한 가입 방지
